# My First Project

はじめてのGitHubプロジェクトへようこそ！

## 概要

このリポジトリは、GitHubの基本的なワークフロー（ブランチ作成、コミット、プルリクエスト）を学ぶためのプロジェクトです。

## せどり収支管理アプリ

`index.html` を開くだけで使える、せどりの収支を一元管理するWebアプリです。ビルドやサーバーは不要で、データはブラウザの localStorage に保存されます。

### 使い方

```bash
# ブラウザで直接開く
open index.html   # macOS
xdg-open index.html   # Linux
```

### 主な機能

- **商品管理**: 商品名・仕入先・仕入額・仕入日・ステータス（在庫／出品中／売却済み）を記録
- **収支記録**: 販売額・手数料・送料に加え、獲得ポイント（楽天ポイント・PayPayポイント等）も記録可能。「現金利益」と「ポイント込み利益」を分けて自動計算
- **複数プラットフォーム対応**: メルカリ・Amazon・ヤフオク・楽天など、プラットフォームは自由に追加・削除可能
- **ダッシュボード**: 今月の売上／利益、在庫状況、直近12ヶ月の月次グラフ、プラットフォーム別売上構成、年次サマリーを表示
- **費用管理**: 固定費（Amazon大口手数料・Keepa・その他ツール）と変動費（梱包費・納品送料・その他費用）を分けて記録。利用日・費用項目・詳細/MEMO・金額を管理し、今月/累計の費用合計を集計
- **データ管理**: JSON形式でのエクスポート／インポート（バックアップ・機種変更時に利用）
- **Excelからの一括登録**: CSVテンプレートをダウンロードし、Excelで入力・CSV保存したファイルを読み込むことで、既存のExcel管理データをまとめて取り込める

すべてのデータはブラウザ内（localStorage）にのみ保存されるため、別端末で使う場合や消去のリスクに備える場合は、データ管理タブからJSONエクスポートでバックアップを取ってください。

## はじめ方

```bash
# リポジトリをクローン
git clone https://github.com/shinjio0176-cloud/-my-first-project.git

# ディレクトリに移動
cd -my-first-project
```

## コントリビューション

1. ブランチを作成する (`git checkout -b feature/your-feature`)
2. 変更をコミットする (`git commit -m 'Add some feature'`)
3. ブランチをプッシュする (`git push origin feature/your-feature`)
4. プルリクエストを開く
